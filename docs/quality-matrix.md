# Diagnostic Quality Matrix
**Author:** Jerome Nuwabaasa, MTIC
**Purpose:** Defines when a value chain diagnostic is ready to publish. The review agent checks every completed diagnostic against these rules before it is accepted. A diagnostic that does not pass ALL mandatory checks is returned for revision — it does not proceed to publication.

**Standard:** This is a decision-grade diagnostic for the Government of Uganda and UNIDO. It must be actionable by the Commissioner, investable by private sector, and credible to development partners. "Good enough" means every check below passes.

---

## HOW TO USE THIS MATRIX

The review agent scores each value chain diagnostic on three levels:
- **MANDATORY** — must pass 100%. One fail = entire diagnostic is returned.
- **EXPECTED** — should pass. More than 2 fails = diagnostic is returned.
- **ADVISORY** — best practice. Noted but does not block publication.

---

## MANDATORY CHECKS ✓ / ✗
*All must pass. Zero exceptions.*

### M1 — Structure
- [ ] All 6 sections present: Value Chain Map, Current State, Constraints, Market Assessment, Prioritized Products, Priority Action Matrix
- [ ] Flows in diagnostic order: current state → constraints → market → solutions (not solutions before diagnosis)
- [ ] Chapter opens with a clear statement of why this chain matters for the Tenfold Growth Strategy
- [ ] References consolidated at end of report — no per-chapter reference lists
- [ ] Abbreviations defined on first use

### M2 — Current State
- [ ] Installed capacity figure present with unit and source
- [ ] Capacity utilisation rate (%) present with source
- [ ] At least 5 key companies named with location and phase operated
- [ ] Import value (USD) present for the primary HS code, sourced from ITC TradeMap
- [ ] Export value (USD) present for the primary HS code, sourced from ITC TradeMap
- [ ] Employment figure present (direct jobs minimum)
- [ ] Value chain map present showing Uganda's position per phase (strong / emerging / gap)
- [ ] Critical gap in the chain explicitly identified

### M3 — Data Standards
- [ ] UBOS is the primary source for all GDP, national output, and employment figures — no World Bank or IMF data used where UBOS data is available
- [ ] All trade figures sourced from ITC TradeMap with HS code cited
- [ ] No figure appears without a source citation
- [ ] No data older than 2022 used without flagging it as such and noting why more recent data is unavailable
- [ ] GDP baseline used is 15.2% (2024) — not 16%

### M4 — Market Analysis
- [ ] All four market levels present: Uganda domestic, EAC/regional, continental (Africa), global
- [ ] 2040 Uganda projection present — uses tenfold growth logic (not a simple ×10 multiplication)
- [ ] Tenfold growth logic explicitly explained for Uganda projection (what structural changes drive demand)
- [ ] BAU growth projection present for EAC, continental, and global markets
- [ ] Uganda's current market share stated for at least one market level

### M5 — Constraints
- [ ] Minimum 5 constraints identified
- [ ] Every constraint has a weight/severity score (Critical / High / Medium / Low)
- [ ] Every constraint has at least one data point or statistic supporting it
- [ ] Cross-cutting constraints flagged for the cross-cutting chapter
- [ ] Most critical constraint is elaborated in sufficient depth that a decision-maker understands it without additional context

### M6 — Prioritized Products
- [ ] Exactly 3–4 priority products identified (not fewer, not more)
- [ ] Each priority product has a composite score from the 5-criterion framework
- [ ] Rationale for each priority product is explicit
- [ ] At least one product is explicitly deprioritized with a clear reason
- [ ] Prioritization rationale connects to the Tenfold Growth Strategy — not just current market size

### M7 — Priority Action Matrix
- [ ] All three timeframes present: quick wins (0–12 months), policy reforms (1–3 years), investment (3–5 years)
- [ ] Minimum 2 actions per timeframe
- [ ] Each action has a named responsible institution
- [ ] Each action has an investment estimate or "policy — no capital cost"
- [ ] At least one quick win is actionable within 6 months with existing institutions

### M8 — Greening & Sustainability
- [ ] Dedicated greening/sustainability section present
- [ ] Current sustainability status described
- [ ] At least 2 upcoming green solutions identified
- [ ] Green dimension framed as a competitiveness enabler (not just a compliance requirement)

### M9 — Tenfold Growth Alignment
- [ ] Opening section explicitly frames the chain in the context of the Tenfold Growth Strategy
- [ ] Priority products selection argues for what gets Uganda to tenfold growth sustainably
- [ ] Project profiles in the action matrix are: (a) strategic for tenfold, (b) include quick wins, (c) incorporate green/sustainability

---

## EXPECTED CHECKS ✓ / ✗
*Maximum 2 fails before diagnostic is returned.*

### E1 — Depth and Credibility
- [ ] Current state section includes company-level data (not just sector-level aggregates)
- [ ] Regional benchmarks present for at least 3 constraints (e.g. Uganda energy tariff vs Kenya, Tanzania)
- [ ] Value chain map shows all possible end products and sub-segments (not just a linear phase list)
- [ ] Per capita consumption figure present (where relevant) to contextualise market size

### E2 — Visuals and Presentation
- [ ] At least 3 tables present in the chapter
- [ ] Market data presented in a table (not only prose)
- [ ] Constraints presented with a severity/weight indicator visible to the reader
- [ ] Status report (sector snapshot) present as an annex — designed to be extracted and shared independently

### E3 — Completeness
- [ ] No `DATA NEEDED` markers remaining in the published chapter
- [ ] No placeholder text (e.g. "TBD", "to be confirmed", "[insert figure]")
- [ ] Investment portfolio chapter covers this chain with at least one project profile
- [ ] Executive summary reflects this chain's key finding

### E4 — Consistency
- [ ] Figures in this chapter are consistent with the sources-of-truth dashboard
- [ ] Priority products in this chapter match the prioritized products recorded in the dashboard
- [ ] 2040 projections in this chapter match the projections in the dashboard

---

## ADVISORY CHECKS ✓ / ✗
*Best practice. Noted in review but does not block publication.*

### A1 — Writing Quality
- [ ] Chapter reads as compelling from first paragraph to last — not as a bureaucratic inventory
- [ ] Each section flows logically from the previous (diagnostic sequence maintained throughout)
- [ ] Technical terms explained on first use for a non-specialist reader

### A2 — Linkages
- [ ] Cross-chain linkages noted (e.g. steel → automotive, plastics → petrochemicals)
- [ ] Circular economy connections noted (e.g. steel slag → cement, bagasse → power)

### A3 — Investment Readiness
- [ ] At least one project profile is written to a standard that could be shared with a financier
- [ ] CAPEX/OPEX figures labelled as "indicative" where not formally costed

---

## SCORING SUMMARY

| Level | Checks | Pass threshold |
|---|---|---|
| MANDATORY | M1–M9 | 100% — one fail returns the diagnostic |
| EXPECTED | E1–E4 | ≥ 14 of 16 checks |
| ADVISORY | A1–A3 | No threshold — logged for improvement |

**Overall verdict:** READY TO PUBLISH / RETURN FOR REVISION

---

## REVISION INSTRUCTIONS (for review agent)
When returning a diagnostic for revision:
1. List every failed check by code (e.g. M2, E3-2)
2. For each failure, state exactly what is missing or wrong
3. For mandatory failures, state which section of the schema covers the missing field
4. Do not approve partial passes on mandatory checks under any circumstances
